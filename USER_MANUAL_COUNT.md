## 模块概述

库存盘点模块是仓库管理系统的核心功能，用于定期检查商品库存数量与系统记录是否相符。通过盘点，可以：
- 发现库存差异（盘盈/盘亏）
- 及时调整库存记录
- 完整记录变更历史，便于审计
- 支持明盘与暗盘两种方式，适应不同的盘点要求

### 核心特性

| 功能 | 说明 |
|-----|------|
| **明盘/暗盘** | 明盘：录入时展示系统库存，便于核对；暗盘：隐藏系统库存，确保独立清点 |
| **快照机制** | 创建盘点单时自动记录系统库存、锁定、冻结等参数，用作对比基准 |
| **批量导入** | 支持Excel/CSV模板导入，提高效率 |
| **差异暂挂** | 生成差异单时自动冻结差异数量，防止其他业务误用 |
| **审核流程** | 差异单需盘库管理员审核，支持强制通过（允许负可用） |
| **完整审计** | 所有操作都生成日志，可追溯 |

## 操作流程

### 第一步：创建盘点单

#### 访问创建页面
点击菜单 **盘点管理** > **创建盘点单**

#### 填写基本信息

1. **选择盘点方式** ⭐ 必填
   - **明盘**：适用于正常盘点。系统会在录入实盘数时展示当前的物理库存、锁定、冻结、可用等数据，辅助核对
   - **暗盘**：适用于严格盘点或防冒对账。系统隐藏所有库存参数，确保盘点人员完全独立清点

2. **选择盘点范围** ⭐ 必填
   - **全部库存**：对所有库存进行盘点
   - **按库位**：选择特定库位进行盘点（如仅盘A区库位）
   - **按分类**：盘点特定商品分类的库存
   - **按商品**：盘点特定商品

   > **提示**：根据实际业务需要选择，系统会自动筛选出范围内的库存

3. **计划盘点日期** ⭐ 必填
   - 选择计划执行盘点的日期
   - 可与实际盘点日期不同

4. **备注** 可选
   - 填写本次盘点的说明信息，如"半年度例行盘点"

#### 创建盘点单

点击 **创建盘点单** 按钮后，系统会：
1. ✓ 创建盘点单记录
2. ✓ 自动为所选范围内的所有商品生成库存快照
3. ✓ 快照内容包括：物理库存数、锁定数量、冻结数量、可用数量

**系统提示**："盘点单 CT20250319001 创建成功，共 87 个商品"

> **注意**：快照一旦创建，后续即使库存有变，也以快照为基准计算差异，确保一致性

---

### 第二步：录入实盘数量

#### 方式一：逐条录入

1. 在盘点单详情页面，查看盘点明细列表
2. 对于**明盘模式**，左侧会显示系统库存数据，便于核对
3. 在 **实盘数量** 列输入实际清点结果
4. 按下回车键或点击其他行自动保存，系统会立即计算差异

**示例**：
```
商品编码   名称      库位    快照数  锁定  冻结  可用  实盘数  差异
P001      商品A    A-01-01  100    10   5    85   105    +5(盘盈)
P002      商品B    A-01-02  50     5    2    43   48     -2(盘亏)
```

#### 方式二：批量导入

**适用场景**：盘点项数众多时，可先在Excel中汇总实盘数据，然后批量导入

**步骤**：

1. 点击 **导出模板** 按钮，下载包含所有盘点项目的Excel模板

2. 打开Excel模板，填写 **counted_qty** 列：
   ```
   sku        product_name  spec   location  snapshot_qty  counted_qty
   SK01001    商品A        10x10  A-01-01   100           105
   SK01002    商品B        20x20  A-01-02   50            48
   SK01003    商品C        30x30  A-01-03   30            30
   ```
   
   > **注意**：
   > - 必须包含 `sku` 和 `counted_qty` 两列
   > - `counted_qty` 必须是非负整数
   > - 不要更改 sku 列，否则匹配会失败

3. 保存Excel文件

4. 返回盘点单详情页，点击 **批量导入** 按钮

5. 选择保存好的Excel文件，点击 **导入**

6. 系统显示导入结果：
   ```json
   ✓ 导入成功！
   更新 87 条记录
   
   错误详情：
   - 第5行：SKU SK01099 - 不在当前盘点范围内
   ```

### 第三步：预览与调整

#### 查看差异统计

在盘点单详情页，系统实时显示进度：
```
商品总数：100     已盘数：95     有差异：12     进度：95%
```

#### 预览差异明细

点击 **预览差异** 按钮，系统分类显示：

1. **盘盈明细** (绿色)
   ```
   商品编码  名称      库位      快照  实盘  盘盈  原因
   P001     商品A    A-01-01   100   105   +5   发现漏录
   ```

2. **盘亏明细** (红色)
   ```
   商品编码  名称      库位      快照  实盘  盘亏  原因
   P002     商品B    A-01-02   50    48    -2   货品损耗
   ```

3. **无差异明细** (灰色)
   - 实盘数量等于快照数量的商品

4. **未实盘警告** (黄色) ⚠️
   - 若有未录入实盘的商品，系统会提示
   - 可返回详情页继续补充，或忽略继续生成差异单

#### 修改实盘数量

若需要修改已录入的实盘数量：

1. 返回盘点单详情页
2. 点击对应商品的 **实盘数量** 输入框
3. 修改数值，按回车键保存
4. 系统自动重新计算差异
5. 若需再次预览，点击 **预览差异** 刷新

---

### 第四步：生成差异单

#### 生成流程

1. 在差异预览页面，点击 **生成差异单并冻结库存** 按钮

2. 系统自动：
   - ✓ 生成盘盈单（如有盘盈项）
   - ✓ 生成盘亏单（如有盘亏项）
   - ✓ **冻结差异数量**到库存中

3. 系统提示："差异单已生成并已冻结差异数量，等待审核..."

#### 理解"冻结"概念

**冻结**是为了防止其他业务在审核期间误用有差异的库存：

- **盘亏冻结**：将差异数量冻结，防止出库等业务操作
  ```
  例：商品P002盘亏2件
  冻结前：物理库存50，冻结数0，可用50
  冻结后：物理库存50，冻结数2，可用48
  ```

- **盘盈冻结**：标记新增的库存，防止未审核就使用
  ```
  例：商品P001盘盈5件
  冻结前：物理库存100，冻结数5，可用95
  冻结后：物理库存100，冻结数10，可用90
  ```

> **重要**：冻结不影响物理库存，只影响可用数量。实际库存调整延迟到审核通过时。

---

### 第五步：审核差异单（仅限管理员）

#### 访问差异单列表

点击菜单 **盘点管理** > **差异单管理**

默认显示 **待审核** 的差异单

#### 审核单个差异单

1. 点击差异单号进入详情页

2. 查看差异明细，确认无误

3. 点击 **审核通过** 或 **驳回** 按钮

#### 审核通过

**操作步骤**：

1. 检查差异明细，特别是金额较大的项目
2. 若需，勾选 **强制通过（允许负可用）** 复选框
3. 点击 **审核通过** 按钮

**库存调整规则**：

- **盘盈单**：增加物理库存
  ```
  调整前：物理库存100，锁定10，冻结5，可用85
  调整后：物理库存105，锁定10，冻结0，可用95
  ```

- **盘亏单**：减少物理库存，同时释放冻结
  ```
  调整前：物理库存50，锁定5，冻结2（来自盘点冻结），可用43
  调整后：物理库存48，锁定5，冻结0，可用43（可用数不变！）
  ```

#### 合理性校验 ⚠️

审核时，系统会检验：**调整后的物理库存 ≥ 锁定数量 + 冻结数量**

If 调整后 物理库存 < 锁定+冻结，系统给出警告：

```
⚠️ 警告：调整后物理库存小于锁定与冻结之和，可用数量为负。

当前：物理库存30，锁定20，冻结5
调整差异：-15（盘亏）
调整后：物理库存15，锁定20，冻结5
计算：15 < 20+5 ❌ （可用 = -10）

处理选项：
[ ] 强制通过（记录在案，便于审计）
[ ✓ ] 驳回（返回盘点环节重新核实）
```

**常见原因**：
- 盘点数据有误（如盘亏数量过大）
- 锁定数量可能不准确（需反查出库订单）
- 库存多货主情况下数据混乱

#### 驳回

**操作步骤**：

1. 输入驳回原因，如"盘亏数据待重新核实"
2. 点击 **驳回** 按钮

**驳回后的处理**：

- ✓ 差异单状态更新为"已驳回"
- ✓ 库存冻结全部释放
- ✓ 盘点人员可返回盘点单修改实盘数量
- ✓ 重新生成差异单后重新审核

## 常见场景

### 👨‍💼 场景 1：定期全库盘点

**目标**：每月月底对全仓库存进行全面清查

**操作**：
1. 创建盘点单，选择 **明盘** 和 **全部库存**
2. 员工分组逐区盘点，录入数据
3. 汇总后预览差异
4. 管理员审核通过
5. 库存自动调整

**时间成本**：约 4-6 小时（500件商品）

---

### 👨‍💼 场景 2：特定库位盘点

**目标**：A区因搬运发生了商品位置调整，需重点盘点确认

**操作**：
1. 创建盘点单，选择 **明盘** 和 **按库位** 筛选 A-01 到 A-05
2. 员工仔细逐件清点
3. 现场发现差异较大，暂不生成差异单
4. 返回详情页修改实盘数量，重新对账
5. 确认无误后生成差异单，管理员审核

---

### 👨‍💼 场景 3：跨部门对账

**目标**：财务部要求严格盘点某分类商品，避免数据作假

**操作**：
1. 创建盘点单，选择 **暗盘** 和 **按分类** 筛选"电子产品"
2. 财务人员独立清点，不查看系统库存
3. 完成后导出实盘数据交仓库核对
4. 反差意见后双方确认最终实盘数
5. 批量导入实盘数据
6. 管理员审核并调整库存

---

### 👨‍💼 场景 4：批量导入盘点数据

**目标**：盘点数据从纸质清单录入系统

**操作**：
1. 从Excel清单整理实盘数据（sku, counted_qty）
2. 下载盘点模板
3. 复制粘贴 sku 和实盘数到模板
4. 批量导入文件
5. 系统提示成功 88 条，失败 2 条（sku 不匹配）
6. 手工修正失败行或逐条补充
7. 预览差异并生成差异单

## 常见问题

### Q1：盘点期间如果有新的入出库，会影响盘点吗？

**A**：不会。系统以快照为基准，盘点期间发生的库存变动不影响差异计算。

例：
```
快照时间 09:00    物理库存 100
   ↓
09:15 出库 10件（系统库存变为 90）
   ↓
14:00 完成盘点，实盘 98 件
   ↓
差异 = 98 - 100 = -2（盘亏）
不受中间出库 10 件的影响
```

若期间库存变动过大，可以**手动刷新快照**，但需确认覆盖原快照。

---

### Q2：什么时候需要勾选"强制通过"？

**A**：当调整后物理库存小于锁定+冻结的和时。

不勾选 ❌：系统会拒绝审核，提示需驳回重新盘点
勾选 ✓：允许审核通过，但会记录日志备查

**示例**：
```
当前状态：
  物理库存 30
  锁定 20（出库订单待执行）
  冻结 5

盘亏要调整 -15

调整后：物理库存 15，但 15 < 20+5，可用为 -10

此时就需强制通过。原因可能是：
- 出库订单数据不准
- 锁定与实际不符
需要后续排查
```

---

### Q3：暗盘模式下员工能看到什么？

**A**：暗盘模式下，员工在录入实盘时看到：

✓ 可以看：商品编码、名称、规格、库位、批次号、快照数量
✗ 看不到：系统的锁定数量、冻结数量、可用数量

这样可确保盘点结果不受系统库存数据的影响，更客观。

---

### Q4：哪些角色可以进行盘点操作？

**A**：需具备 `inventory_manage` 权限的用户。

- **创建/录入实盘**：普通员工 + 管理员
- **生成差异单**：普通员工 + 管理员
- **审核差异单**：仓库管理员及以上
- **强制通过**：仓库管理员及以上

---

### Q5：导入失败了怎么办？

**A**：系统会返回错误列表，常见原因：

| 错误类型 | 原因 | 处理 |
|--------|------|------|
| sku 不存在 | 填错商品编码 | 确认 sku 正确，或创建新商品 |
| 数量格式错误 | 非数字或负数 | 检查 Excel 中的数据格式 |
| 不在当前盘点范围 | sku 不属于选定范围 | 扩大盘点范围或手工补充 |

修正后可重新导入，成功的行会更新，失败的行会重新处理。

---

### Q6：能否中途修改快照？

**A**：可以，但需谨慎。

点击 **刷新快照** 会覆盖原快照并生成新的快照。系统会提示：
```
⚠️ 警告：此操作会覆盖原有快照数据。
原快照时间：2025-03-19 09:00
新快照时间：2025-03-19 14:00

提醒：库存若已发生变动，差异会重新计算。

☐ 我已确认，继续刷新
```

**使用场景**：
- 盘点进行中发生重大入库导致系统库存大幅变化
- 发现快照数据有误（如手动调整过库存）
- 需要与最新库存对齐

---

### Q7：驳回后如何重新盘点？

**A**：
1. 差异单被驳回后，库存冻结会自动释放
2. 返回盘点单详情页
3. 找到有问题的商品，修改 **实盘数量**
4. 系统重新计算差异
5. 返回预览差异页面，重新 **生成差异单**
6. 管理员重新审核

---

### Q8：为什么有的商品显示"不在当前盘点范围"？

**A**：说明该商品不符合创建盘点单时设置的范围。

例：
- 盘点范围选择"A区库位"，但该商品在 B区
- 盘点范围选择"电子产品分类"，但该商品属于其他分类

**处理**：
- 方案1：扩大盘点范围重新创建
- 方案2：逐条在盘点单详情页补充该商品（如果系统支持）
- 方案3：将该商品挪到指定范围后重新盘点

---

### Q9：生成的库存变更日志在哪查看？

**A**：
1. 进入 **库存管理** > **库存列表**
2. 点击具体商品的 **详情**
3. 下方会显示 **库存变更日志**，记录所有调整历史
4. 可看到盘点相关的变更，包括单号、变动量、时间等

---

### Q10：盘点单生成后还能删除吗？

**A**：
- **草稿状态**：可删除（联系管理员）
- **进行中/已完成**：不建议删除，会破坏审计链

若确实需要作废，应该：
1. 创建新的盘点单进行"反向盘点"来抵消
2. 或者驳回已生成的差异单，后再删除

## 数据字典

### 状态定义

**盘点单状态**：
- **draft** (草稿)：刚创建，可编辑范围和快照
- **in_progress** (进行中)：已开始录入实盘数据
- **completed** (已完成)：已生成差异单，等待审核
- **closed** (已关闭)：审核完成，调整已生效

**差异单状态**：
- **pending** (待审)：已生成，等待管理员审核
- **approved** (已通过)：审核通过，库存已调整
- **rejected** (已驳回)：审核不通过，允许修改后重新生成

### 字段说明

| 字段 | 说明 | 示例 |
|-----|------|------|
| count_no | 盘点单号 | CT20250319001 |
| variance_no | 差异单号 | VD20250319001 |
| snapshot_qty | 快照库存数量 | 100 |
| counted_qty | 实盘数量 | 105 |
| variance_qty | 差异数量 | 5（正数为盘盈，负数为盘亏） |
| snapshot_locked | 快照时锁定数 | 10 |
| snapshot_frozen | 快照时冻结数 | 5 |
| available | 可用数量 | 85 = 100-10-5 |

### 色彩编码

- 🟢 **绿色**：盘盈（好消息，库存增加）
- 🔴 **红色**：盘亏（需要关注，库存减少）
- ⚪ **灰色**：无差异（符合预期）
- 🟡 **黄色**：警告/未完成（需要处理）
- 🔵 **蓝色**：待审（需要管理员处理）
